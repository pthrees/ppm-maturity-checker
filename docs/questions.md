# 設問リスト

## 回答方式

各設問に対して以下の2軸で回答する。

### 成熟度（Maturity）— Slider入力

| 値  | ラベル   | 説明                           |
| --- | -------- | ------------------------------ |
| 0   | 未実施   | まだ取り組んでいない           |
| 1   | 一部実施 | 一部の部門・プロジェクトで実施 |
| 2   | 標準化済 | 組織として標準的に運用している |
| 3   | 最適化   | 継続的に改善・最適化されている |

### 重要度（Importance）— Radio選択

| 値  | ラベル |
| --- | ------ |
| 1   | 低     |
| 2   | 中     |
| 3   | 高     |

---

## カテゴリA: 稼働管理

> リソースの稼働状況と負荷の可視化

| ID  | タイトル       | 設問内容                                                                |
| --- | -------------- | ----------------------------------------------------------------------- |
| A1  | 工数入力率     | 全メンバーが日次・週次で正確に工数を入力し、実態が把握できているか      |
| A2  | 予実一元把握   | 計画工数と実績工数の乖離をリアルタイムで比較・確認できる仕組みがあるか  |
| A3  | PM負荷見える化 | PM/PL層の管理負荷やオーバーワーク状況を組織として定量的に把握しているか |

---

## カテゴリB: スキル・配員

> 最適な人材配置とスキル管理

| ID  | タイトル       | 設問内容                                                                       |
| --- | -------------- | ------------------------------------------------------------------------------ |
| B1  | スキルアサイン | 個人のスキルレベルに基づいた科学的なアサイン（配員）が行われているか           |
| B2  | 炎上防止       | 特定のリソースへの依存度やリスクを事前に検知し、炎上を未然に防ぐ手立てがあるか |
| B3  | 要件定義(型)化 | 要件定義や設計のプロセスが標準化され、属人化を排除できているか                 |

---

## カテゴリC: 収益性管理

> プロジェクトの予実管理と採算性

| ID  | タイトル         | 設問内容                                                                             |
| --- | ---------------- | ------------------------------------------------------------------------------------ |
| C1  | 見積り根拠標準化 | 見積もりの前提条件や計算ロジックが組織標準として確立されているか                     |
| C2  | 乖離要因特定     | 予実乖離が発生した際、その真因（見積もりミス、仕様変更、生産性等）を特定できているか |
| C3  | 撤退基準明確化   | 赤字プロジェクトやリスク案件に対する撤退・縮小基準が明確に運用されているか           |

---

## カテゴリD: プロセス成熟度

> 組織的な管理プロセスとガバナンス

| ID  | タイトル           | 設問内容                                                                                  |
| --- | ------------------ | ----------------------------------------------------------------------------------------- |
| D1  | RAID管理標準化     | リスク(Risk)、前提(Assumption)、課題(Issue)、依存(Dependency)の管理手法が統一されているか |
| D2  | 経営ダッシュボード | 経営層が必要とするプロジェクトポートフォリオ全体の状況が即座に可視化されているか          |
| D3  | リソース調整会議   | 部門を超えたリソース調整や優先順位決定を行う会議体やプロセスが機能しているか              |
